Challenges:
Since the original system on 3/27/2026 removed duplicate items, this meant that once the items objectName+key were in there, it wouldn't work again. UItemGetPickup_C has 6 instances initialized at gametime (fucking stupid I know). This is how they allow overlapping widgets when you collect multiple items... yeah great system. Anyway, this meant that once the duplicates where full I was fucked. Instead I spent hours refactoring, trying to access VTables on these objects, everything. The main issue is this, ProcessEvent runs the following functions: OnPreConstruct -> Construct -> Tick <-> OnEventPressed, where Tick and OnEventPressed are constantly getting called while the widget is displayed. Well, I checked on Construct 2 nights ago, and it gives uninitialized data, which isn't useful... but what if I waited until it was?! Then like that I integrated a system and it fucking works. That was my main challenge, but now we need to have a ItemPickupManager of some description and then have that contain a dependency injected Client, but we will worry about that later. Its the small victories that make a massive victory.
